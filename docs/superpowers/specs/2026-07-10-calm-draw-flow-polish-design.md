# 抽牌节奏与解码页精修设计

日期：2026-07-10

## 目标

在不依赖 Figma、不改变现有蓝金星空视觉语言的前提下，解决三个明确问题：牌位能量光叠在牌背上、连续抽牌过于急促、结果页卡牌过大且服务配置错误破坏沉浸感。

## 交互方案

- 非最后一张牌确认后进入 `PLACING`，停留 800ms，让落牌能量有完整的收束过程，再恢复选牌。
- 最后一张牌确认后保留 1100ms 的静默确认，再进入统一揭示。
- 统一揭示使用 1800ms 翻牌和 900ms 结果沉降，总计 2700ms 后进入解码页。
- `PLACING` 阶段屏蔽选牌输入，避免用户在视觉尚未完成时连续触发下一张牌。

## 视觉方案

- 选牌和落牌阶段，牌阵提示层放到牌环后方；只有统一揭示时才升到牌环前方。
- 选牌指针与候选牌光晕缩小、降亮度，不再用大面积白金光覆盖牌背图案。
- 结果页三张牌宽度从 `210–310px` 缩到 `160–210px`，牌阵区域高度收紧，并把更多垂直空间交给综合解读。
- 五张牌同步小幅缩小，维持星阵构图，不改变牌位关系。
- DeepSeek 未配置、密钥或额度异常等内部服务信息统一转为面向用户的文案，不暴露部署配置。

## 边界

- 不修改抽牌随机性、正逆位逻辑、手势识别阈值和接口协议。
- 不重做页面品牌风格，不引入新依赖。
- 保持页面在 `100dvh` 内，移动端仍可使用。

## 验收标准

1. 抽牌时牌位能量与标签不会覆盖牌背主体。
2. 每次落牌后有可感知的停顿，最后确认到揭示不再瞬间跳转。
3. 三张牌结果页卡牌明显缩小，综合解读区获得更多空间。
4. 未配置 DeepSeek 时不再显示“密钥”等内部配置词。
5. 测试、类型检查、Lint 和生产构建通过。
