# 牌库紧凑布局设计

**日期：** 2026-07-16
**范围：** `/library` 与 `/library/[cardId]` 的界面层；不改数据、筛选逻辑、URL 行为、抽牌、手势抽牌、聊天或 DeepSeek。

## 目标

让牌库首页在桌面首屏更快出现搜索、筛选与卡牌，让详情页以“左侧牌面、右侧信息”的方式阅读。优先减少空白和重复的大标题，而不通过压低正文可读性来强行塞满所有内容。

## 牌库首页

- 保留顶部品牌与返回首页入口。
- 将当前大幅“原创塔罗牌库”与世界观说明替换为紧凑档案栏：小标题“牌库”、牌组数量与简短状态同行呈现，不保留大段说明文。
- 在桌面端将搜索、阿尔卡纳筛选、花色筛选和结果数组织为紧凑控制区；卡牌网格紧随其后。
- 继续使用现有 URL 驱动的筛选状态、搜索输入、筛选按钮与清空行为，不新增服务端请求。

## 详情页

- 1024px 及以上使用固定宽度的左侧牌面栏和自适应右侧内容栏；牌面在可用高度内粘性停留，阅读时不丢失视觉锚点。
- 右侧依次保留牌名、关键词、正逆位切换、场景记录、人物/地点/符号与前后牌导航；缩小大标题、分区标题和区块间距。
- 宽屏下将人物、地点、符号改为三列紧凑信息项，避免当前纵向时间线造成大量空白。
- 正逆位标签页保留键盘左右键、ARIA 角色和现有焦点样式；前后牌导航及面包屑保持不变。
- 768px 以下恢复单列顺序，卡面不粘性，筛选控件允许换行或横向滚动，确保不溢出。

## 视觉基调

沿用当前深靛蓝、低饱和金色、青绿色档案索引和衬线标题。视觉注册为“紧凑档案台”：控制密度高、层级清楚、留白服务于阅读而非装饰。

## 验证

- 运行现有 `npm test`、`npx tsc --noEmit`、`npm run lint`、`npm run build`。
- 在桌面与移动宽度检查首页首屏、筛选控件、详情双栏/单栏切换、文字折行、标签页与前后导航。
- 不将“所有内容永不滚动”当作硬性承诺；当故事文案超过正常长度时，正文应自然继续向下，而非缩小到不可读。
