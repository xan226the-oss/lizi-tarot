# 手势抽牌极简沉浸与性能优化设计

日期：2026-07-10

## 目标

在保留三张牌、五张牌、后台手势识别和 DeepSeek 综合解读的前提下，同时解决以下问题：

1. 删除与背景割裂的圆环、方框、仪式线和人物摄像头画面。
2. 结果页不再重复显示两次用户问题。
3. 牌群与背景通过共享光线、雾气和景深形成同一空间。
4. 旋转、洗牌和选牌减少掉帧，不再一卡一卡。
5. 隐藏摄像头画面后，用户仍能知道手势识别是否正常。

## 已确认方向

用户选择 A「极简沉浸」：

- 只保留深蓝星空、塔罗牌、必要文字和柔和金色亮光。
- 不保留中央圆环、外框、罗盘、侧边装饰、祭坛线或牌位方框。
- 摄像头继续在后台识别手势，但人物画面不出现在界面中。
- 牌与背景使用共享蓝金雾光和外围轻景深融合，不通过新增几何装饰融合。

## 根因分析

### 画面分层

当前卡牌使用接近黑色的不透明底、清晰金边、多层投影和独立发光；背景则是柔软、低对比的蓝色星雾。背景光没有映入牌边，牌的金光也没有扩散至周围空间，因此卡牌像贴在背景上的独立图层。

此外，牌群、背景圆环、祭坛、牌位框和提示框分别拥有自己的边界与动画，形成多层相互竞争的视觉容器。

### 掉帧

当前高频路径同时包含：

- `pointer / orbitOffset / orbitSpeed / selectedCardId` 经页面 reducer 每帧更新，引发整页重新渲染。
- 约 260 个粒子持续执行 CSS 动画，并随父组件渲染参与协调。
- 15 张卡牌移动时同时使用 transform 过渡、模糊、滤镜、阴影和多重洗牌动画。
- 三层背景圆环、祭坛和其他装饰持续旋转或呼吸。
- MediaPipe 每个视频帧除识别外，还绘制摄像头画面、手部连线和调试状态。

## 最终视觉

### 舞台背景

- 使用现有深蓝星空作为唯一基础背景。
- 删除有明确边界的圆环、椭圆、外框和祭坛。
- 牌群后方增加一层无可见边缘的蓝金径向雾光，覆盖整个扇面，而不是围住单张牌。
- 雾光只缓慢漂移，不做高速旋转。
- 粒子数量降至约 100 个，粒子更稀疏、更小，不形成独立前景层。

### 牌群融合

- 降低卡背金边、黑底和投影的对比度。
- 给整组牌加入统一的环境蓝光染色，避免每张牌拥有互不相干的强光圈。
- 外围牌依据深度自然降低亮度和清晰度；中央牌保持最清晰。
- 牌群底部使用柔和接触雾光承托，使牌处在同一个空间，不像悬空贴片。
- 选中牌只增强局部亮度和向外扩散的柔光，不出现矩形发光框。

### 提示与牌位

- 阶段标题和操作提示直接浮在背景上，不使用玻璃方框。
- 未填牌位只显示牌位文字和一个很弱的光点，不显示矩形轮廓。
- 已填牌位使用无边界光雾和少量流动粒子，不使用圆环或贴图。
- 顶部保留返回和模式切换两个功能控件；其余装饰性侧栏、罗盘、角框和品牌图形删除。

### 结果页

- 抽牌阶段顶部显示用户问题。
- 进入 `RESULT` 后，顶部中央只显示小型状态文字“命轮已揭示”，不再重复问题。
- 用户问题只在结果主体中作为唯一大标题出现。
- 三张牌继续等权并列；五张牌继续使用五个星阵位置，但不绘制五芒星线、圆环或额外背景框。
- 删除结果卡内部的大圆形十字图案，改为一个小型星点和短金线。
- DeepSeek 综合解读保留横向区域，但弱化面板边界，以一条分隔线与内容区连接。

## 摄像头与手势反馈

- 保留 MediaPipe 视频输入和手势识别。
- 视频元素缩至不可见的传感器节点，不展示人物画面。
- 删除可见的摄像头面板、手部骨架 Canvas 和 MediaPipe 调试信息。
- 停止每帧绘制手部连线和控制点，只运行识别模型。
- 使用顶部小型状态点和短文字反馈：
  - 金色点：“手势识别中”
  - 弱色点：“请将手放入画面”
  - 错误状态：“摄像头不可用，可切换点击模式”
- 状态只在发生变化时更新，避免新的高频渲染。

## 性能架构

### 高频视觉状态下沉

新增独立的手势舞台运行层，局部持有以下瞬时状态：

- `pointer`
- `orbitOffset`
- `orbitSpeed`
- 当前视觉候选牌

运行层只重绘 `TarotUniverse / CardOrbit`，不让这些值每帧进入页面业务 reducer。

页面 reducer 继续管理：

- 流程阶段
- 问题和牌阵
- 已抽结果
- 已确认抓取牌
- 模式、错误和结果状态

当捏合确认时，运行层把当时的候选牌 ID 传给页面；页面只在确认抽牌或阶段切换时更新。这样保持“输入、视觉、业务”职责分离，同时阻止牌环运动带动整页重绘。

### 动画减负

- `ParticleField` 使用缓存组件，手势页数量降至约 100。
- 卡牌位移不再使用持续追赶目标的 `380ms` transform 过渡。
- 保留现有阻尼算法提供自然惯性，卡牌只执行一层 GPU transform。
- 洗牌只保留一个外层位置动画，删除同时运行的内层旋转和模糊滤镜。
- 运动中的卡牌不使用 CSS blur；景深通过亮度、透明度和轻量阴影表达。
- 删除圆环、祭坛、外框后，同步删除对应的旋转、呼吸、模糊和阴影动画。
- 静态背景、页头和粒子层使用缓存，不参与每一帧的牌群更新。

## 数据流

1. MediaPipe 或点击输入产生指针帧。
2. `GestureController` 把输入转换为视觉运动帧。
3. 手势舞台运行层局部更新牌群位置与候选高亮。
4. 用户确认捏合时，运行层把候选牌 ID 交给页面 reducer。
5. 页面 reducer 记录业务结果并切换阶段。
6. `RESULT` 继续使用完整三张或五张结果请求 DeepSeek 综合解读。

## 不变内容

- 三张牌与五张牌的牌阵含义和数量不变。
- 卡牌尺寸和宽幅扇形方向不缩小。
- 正逆位、牌位和综合解读逻辑不变。
- 手势分类阈值不调整。
- 点击模式继续保留。
- 每日一签和其他页面不在本次范围内。

## 验证标准

### 视觉

1. 抽牌页不存在可见摄像头人物、手部骨架或调试面板。
2. 舞台不存在装饰性圆环、外框、罗盘、祭坛或牌位方框。
3. 选中牌不存在矩形外发光框，只显示柔光和亮度变化。
4. 三张和五张结果页只出现一次用户问题。
5. 三张结果保持等权；五张结果五张牌均完整可见。
6. 卡牌边缘能看到环境蓝金染色，牌群后方光雾没有清晰几何边界。

### 交互与性能

1. 点击模式连续横向旋转 5 秒，不出现规律性停顿或整页闪动。
2. 手势模式连续旋转时人物不显示，识别状态文字可正确变化。
3. 高速旋转时页头、背景与结果区域不随牌环每帧重新渲染。
4. 洗牌、选牌、抓取、统一揭示和重置流程均能完成。
5. 1280×720 与 390×844 下没有页面级横向或纵向溢出。
6. 浏览器控制台无新增错误。
7. `npm test`、`npm run lint`、`npx tsc --noEmit`、`npm run build` 全部通过。

## 风险与缓解

- 隐藏摄像头后无法直接查看手的位置：使用状态点和“请将手放入画面”提示缓解。
- 装饰减少后可能显空：使用共享雾光、环境染色和深度层次补足，不重新添加几何框。
- 去掉长 transform 过渡后运动更直接：保留数学阻尼和速度衰减，确保自然跟手。
- 高频状态下沉可能影响确认牌 ID：捏合确认时显式传递运行层当前候选 ID，并覆盖点击与手势两条验证流程。

## 非目标

- 不引入 Three.js、R3F 或新的动画依赖。
- 不改变 DeepSeek API 契约。
- 不重新设计问题输入页或牌阵选择页。
- 不增加登录、数据库或历史记录功能。
