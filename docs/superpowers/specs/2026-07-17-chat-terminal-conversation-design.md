# 无气泡沉浸式单聊界面设计

**日期：** 2026-07-17
**状态：** 用户已确认设计
**范围：** 仅改善 `/chat/[cardId]` 的单聊呈现与输入体验；不改 API、人格数据、安全逻辑、持久化或群聊。

## 目标

把当前传统消息列表与默认白色输入框改为沉浸式单聊。用户和角色的发言以位置、色彩和留白区分，不使用聊天软件式气泡或显眼身份标签；输入文字在深色表面上始终可读。聊天画布还应提供不依赖外部图片的静态分层星空，以及桌面指针移动时低干扰的短暂粒子反馈。

## 布局

- 不设置固定牌面头像、终端栏、在线状态或大标题；页面直接进入对话空间。
- 角色开场语是第一条左对齐消息，使用冷蓝文字、极轻角色名和微光竖线，不使用卡片或气泡边框。
- 用户消息右对齐，仅使用淡紫色文本；不显示“你”或身份标签，也不使用气泡。
- 整个页面是全视口聊天画布，不使用 760px 窄卡片或外框。中央阅读列仅限制每条消息的行宽，对话区域本身占用输入栏以外的所有可视高度。
- 消息区是独立滚动区域，消息间保留显著留白，像同一空间里交替出现的两段思绪；页面本身不应出现大块未被对话利用的空白。
- 输入区固定在浏览器视口底部，不属于中央阅读列；输入采用单一深色半透明输入条，而非大文本框。消息区必须预留与输入区相等的底部安全空间，任何一条消息都不能被固定输入区遮挡；移动端同样适用。
- 两项声明仅在尚未同时确认时显示为紧凑卡片；确认后完全从视图中消失，不持续提醒用户。

## 星空与指针粒子

- 星空由 `Chat.module.css` 的深蓝底色、多个 CSS 径向渐变与低对比度光晕构成；不加载图片、视频、字体或第三方动画库，避免版权和网络依赖。
- `StarfieldParticles` 是只负责视觉反馈的独立客户端 Canvas 层，位于内容之后、消息和输入之前，且始终 `aria-hidden` 与 `pointer-events: none`；它不能影响点击、选择文本、滚动、表单提交或任何聊天状态。
- 仅在具有精确指针且未请求减弱动态时启动。指针移动每次最多补充少量微粒，微粒在很短时间内随透明度下降而消失，并设置严格总数上限；不绘制常驻 Canvas 星空，不做鼠标轨迹长尾，不读取或保存用户数据。
- 触摸/粗指针设备与 `prefers-reduced-motion: reduce` 下不初始化 Canvas、监听器或动画帧，只保留静态星空。没有 `matchMedia`、`ResizeObserver` 或 Canvas 2D 上下文时静默不启用效果，聊天功能照常可用。

## 视觉与可访问性

- 页面保持深蓝星际底色与玻璃层次；不增加固定头像、假终端日志、聊天软件式气泡或过多装饰动画。
- 文本输入区使用深色半透明单行/两行输入条、浅色输入文字、较低对比度 placeholder、金色焦点描边；禁止白色默认输入框。
- 保持当前最小 44px 可操作面积、键盘焦点、`aria-live` 加载提示与 `prefers-reduced-motion` 支持。
- 消息继续只作为普通文本渲染，不能引入 Markdown/HTML 注入。

## 组件边界

- `ChatMessageList`：只负责左右对齐的无气泡纯文本消息、角色微标签和加载提示。
- `SingleCardChat`：只负责全视口聊天层、条件展示的声明卡、沉浸式输入控制和既有请求生命周期。
- `StarfieldParticles`：只负责基于指针坐标的短暂 Canvas 粒子绘制与其完整清理，不接触消息、请求、同意或持久化状态。
- `Chat.module.css`：只负责本次全视口无气泡布局、独立滚动、对比度与响应式样式。

## 验收

1. 浏览器默认白底、黑字输入框不再出现；深色单一输入条中的输入文字与 placeholder 可读。
2. 角色左对齐、用户右对齐，均不使用气泡或显眼身份标签；首条开场也属于角色消息。
3. 两项同意在完成后从视图消失；不显示固定牌面头像、终端栏或在线状态。
4. 不修改 API、消息协议、安全/同意条件或本页内存状态。
5. 390px 宽与桌面宽下无横向溢出；聊天画布填满可视高度，消息区独立滚动；固定输入区始终可见，消息不会被输入区遮挡。
6. 现有聊天合同、类型检查和 lint 继续通过。
7. 桌面精确指针上，移动鼠标可见克制的短暂微粒；不妨碍点击、输入或滚动，且不会无限累积。
8. 移动端、粗指针或 `prefers-reduced-motion: reduce` 下只显示静态星空，不运行指针粒子动画。
